# Описание проекта
Бэкенд веб-приложение блога реализован с использованием Spring Boot, выполняющегося в сервлет-контейнере Tomcat.

Сборка осуществляется с помощью системы сборки Gradle. Приложение написано на Java 21.
База данных в памяти (H2).

Проект покрыт тестами (юнит и интеграционными) с использованием JUnit 5.

### Требования к окружению
- **Java 21+**

### Как запустить и использовать бэкенд
1. Убедитесь, что у вас установлен [Java Development Kit (JDK)](https://www.oracle.com/java/technologies/javase-downloads.html) версии 21 или выше.
2. В корне проекта выполните команду:
```bash
./gradlew bootRun
```
3. После запуска, бэкенд будет доступен по адресу:
`http://localhost:8080/`
4. Для остановки выполните ctrl + c, а затем подтвердите нажатием y.

## Основная функциональность
Бэкенд обеспечивает REST API для взаимодействия с фронтендом ("Мой Блог", реализован на React) и реализует следующие операции:

### Получение списка постов
- **Метод:** GET
- **URL:** `/api/posts?search=filter&pageNumber=1&pageSize=5`
- **Описание:** Возвращает список постов с учетом поиска и пагинации.
- **Ответ:** JSON
```json
{
  "posts": [
    {
      "id": 1,
      "title": "Название поста 1",
      "text": "Текст поста в формате Markdown...",
      "tags": ["tag_1", "tag_2"],
      "likesCount": 5,
      "commentsCount": 1
    },
    ...
  ],
  "hasPrev": true,
  "hasNext": false,
  "lastPage": 3
}
```
---
### Получение конкретного поста
- **Метод:** POST  
- **URL:** `/api/posts/1` (где 1 — ID поста)  
- **Ответ:** JSON
```json
{
"id": 1,
"title": "Название поста 1",
"text": "Текст поста в формате Markdown...",
"tags": ["tag_1", "tag_2"],
"likesCount": 5,
"commentsCount": 1
}
```
---
### Создание нового поста

- **Метод:** POST  
- **URL:** `/api/posts  `
- **Тело** (JSON):
```json
{
"title": "Название поста 3",
"text": "Текст поста в формате Markdown...",
"tags": ["tag_1", "tag_2"]
}
```
- **Ответ:** созданный пост с id, likesCount=0, commentsCount=0:
```json
{
"id": 3,
"title": "Название поста 3",
"text": "Текст поста в формате Markdown...",
"tags": ["tag_1", "tag_2"],
"likesCount": 0,
"commentsCount": 0
}
```
---
### Обновление поста

- **Метод:** PUT  
- **URL:** `/api/posts/3  `
- **Тело:**
```json
{
"id": 3,
"title": "Название поста 3",
"text": "Обновленный текст",
"tags": ["tag_1", "tag_2"]
}
```
- **Ответ:** обновленный пост с актуальными likesCount и commentsCount.
---
### Удаление поста

- **Метод:** DELETE  
- **URL:** `/api/posts/1  `
- **Ответ:** После успешного удаления возвращается статус 200 OK.
---
### Инкремент лайка

- **Метод:** POST  
- **URL:** `/api/posts/1/likes  `
- **Ответ:** Возвращает новое число лайков.
---
### Получение списка комментариев поста
- **Метод:** GET
- **URL:** `/api/posts/{id}/comments`
- **Ответ:** Объект `CommentDto`, содержащий:
    - `id` — идентификатор комментария.
    - `text` — текст комментария.
    - `postId` — идентификатор поста, к которому принадлежит комментарий.
postId — идентификатор поста, к которому принадлежит комментарий.
```json
  [
    {
      "id": 1,
      "text": "Комментарий к посту 1",
      "postId": 1
    },
    {
      "id": 2,
      "text": "Ещё один комментарий к посту 1",
      "postId": 1
    }
  ]
```
---
### Получение конкретного комментария
- **Метод:** GET
- **URL:** `/api/posts/{id}/comments/{comId}`
- **Ответ:** Объект `CommentDto`, содержащий:
    - `id` — идентификатор комментария.
    - `text` — текст комментария.
    - `postId` — идентификатор поста, к которому принадлежит комментарий.
```json
  {
    "id": 2,
    "text": "Ещё один комментарий к посту 1",
    "postId": 1
  }
```
---
### Добавление нового комментария
- **Метод:** POST
- **URL:** `/api/posts/{id}/comments`
- **Тело запроса:**
```json
{
    "text": "Ваш комментарий",
    "postId": 1
}
```
- **Ответ:** Объект `CommentDto`, содержащий:
    - `id` — идентификатор комментария.
    - `text` — текст комментария.
    - `postId` — идентификатор поста, к которому принадлежит комментарий.
Пример ответа:
```json
{
    "id": 3,
    "text": "Комментарий к посту",
    "postId": 1
}
```
---
### Обновление комментария
- **Метод:** PUT
- **URL:** `/api/posts/{id}/comments/{comId}`
- **Тело запроса:**
```json
{
"id": 2,
"text": "Обновленный текст комментария",
"postId": 1
}
```
- **Ответ:** Обновленный комментарий в формате CommentDto.
```json
{
"id": 2,
"text": "Обновленный текст комментария",
"postId": 1
}
```
---
### Удаление комментария
- **Метод:** DELETE
- **URL:** `/api/posts/{id}/comments/{comId}`
- **Ответ:** При успешном удалении возвращается статус 204 No Content.

---

### Добавление изображения к посту
- **Метод:** POST
- **URL:** `/api/posts/{id}/image`
- **Описание:** Загружает изображение для поста с указанным `id`.
- **Параметры запроса:**
    - `image` — файл изображения типа Multipart/form-data.
- **Ответ:**  
  **Статус:** 204 No Content  
  **Описание:** Изображение успешно загружено.

---

### Получение изображения поста
- **Метод:** GET
- **URL:** `/api/posts/{id}/image`
- **Описание:** Загружает изображение, прикрепленное к посту с указанным `id`.
- **Ответ:**  
  **Тип:** `byte[]` (изображение)  
  **Статус:** 200 OK

---
### Обновление изображения поста
- **Метод:** PUT
- **URL:** `/api/posts/{id}/image`
- **Описание:** Обновляет изображение для поста с указанным `id`.
- **Параметры запроса:**
    - `image` — файл изображения типа Multipart/form-data.
- **Ответ:**
    - **Статус:** 204 No Content
    - **Описание:** Изображение успешно обновлено.
- **Пример использования:**  
  Отправка нового файла через форму или инструмент типа Postman.
